# 📚 使用示例

本文档提供了不同场景下的使用示例，帮助你快速上手通用编程Agent。

## 🎮 游戏开发场景

### 示例1: 创建一个简单的贪吃蛇游戏

```
任务: 创建一个贪吃蛇游戏，包含游戏逻辑、食物生成、碰撞检测
```

**Agent执行流程**:
1. 分析需求，规划项目结构
2. 创建游戏主文件 `game.py`
3. 创建蛇类 `snake.py`
4. 创建食物类 `food.py`
5. 创建配置文件 `config.py`
6. 运行测试，确保游戏可运行

### 示例2: 添加游戏功能

## 💡 使用技巧

### 1. 明确任务描述
- ✅ 好的描述: "创建一个Flask REST API，包含用户管理功能（增删改查）"
- ❌ 模糊描述: "做个API"

### 2. 分步骤执行复杂任务
对于复杂任务，可以分步骤执行：
1. 先创建项目结构
2. 再实现核心功能
3. 最后添加测试

### 3. 利用代码质量工具
Agent会自动进行代码质量检查，但你可以明确要求：
- "生成代码后运行语法检查"
- "创建测试文件并运行测试"

### 4. 场景选择
根据任务类型选择合适的场景：
- 游戏开发 → game_dev
- Web应用 → web_dev
- 数据分析 → data_science
- 部署运维 → devops
- 通用任务 → general

## 🎯 最佳实践

1. **先规划后执行**: Agent会自动规划，但你可以在任务中明确要求
2. **测试驱动**: 要求Agent创建测试文件，确保代码质量
3. **代码审查**: 使用代码质量工具检查生成的代码
4. **备份重要文件**: 在修改重要文件前，Agent会自动备份
5. **迭代改进**: 分步骤完成任务，逐步完善功能

## 📝 注意事项

- Agent会尝试自动安装依赖，但某些包可能需要手动安装
- 代码质量工具（ruff, mypy）是可选的，未安装时会跳过相应检查
- 某些操作（如Docker命令）有安全限制，只允许安全命令
- 文件备份会自动添加时间戳，便于恢复
